# Wiki Links Guide

This guide demonstrates Obsidian-style wiki links for connecting notes.

## What are Wiki Links?

Wiki links allow you to link between pages using a simple double-bracket syntax, just like in Obsidian or Roam Research.

## How to Use

### Basic Link
Link to a page by its filename (without .md extension):

- `[[python]]` - Links to python.md
- `[[javascript]]` - Links to javascript.md
- `[[docker]]` - Links to docker.md

### Link by Title
You can also link by the page title:

- `[[microservices]]` - Links to microservices.md
- `[[design-patterns]]` - Links to design-patterns.md

### Custom Link Text
Use the pipe `|` to customize the link text:

- `[[python|Python Programming Guide]]` - Shows "Python Programming Guide" but links to python.md
- `[[react|React Framework]]` - Shows "React Framework" but links to react.md

## Examples in Context

When learning [[python]], you might also want to explore [[javascript]] for web development. Both languages are essential for modern development.

For backend development, understanding [[docker]] and [[kubernetes]] is crucial. You should also learn about [[microservices]] and [[design-patterns]].

Database knowledge is important too - check out [[postgresql]] and [[redis]] for different use cases.

## Benefits

- **Faster linking**: No need to remember exact file paths
- **Automatic updates**: If you rename a file, links still work
- **Bidirectional**: Easy to see what links to what
- **Familiar**: Same syntax as Obsidian, Roam, and other PKM tools

## Tips

- Use lowercase for consistency
- Prefer hyphens over spaces in filenames
- Link liberally - connections create knowledge
- Use custom text when context needs it
